■■ Base de données
5 tables suffisent pour tout le MVP. Les flèches → indiquent les clés étrangères.
users
id (uuid) email full_name created_at
→ Géré par Supabase Auth automatiquement
properties
id user_id → name address type surface rooms rent charges deposit status notes created_at
→ user_id = clé étrangère vers users (isolation des données)
tenants
id user_id → property_id → first_name last_name email phone birthdate entry_date exit_date
rent_amount deposit_paid active
→ Un locataire est toujours lié à un bien
payments
id tenant_id → property_id → period_month period_year amount_expected amount_paid paid_at
payment_method status notes
→ Généré auto chaque mois pour chaque locataire actif
documents
id user_id → property_id → tenant_id → category filename storage_path uploaded_at
→ Fichiers stockés dans Supabase Storage

■ Sécurité (Row Level Security)
Activer le RLS dans Supabase sur chaque table avec la règle :
WHERE user_id = auth.uid()
Chaque propriétaire ne voit que ses propres données, sans code supplémentaire.
